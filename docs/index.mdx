# Meeting Planner — dokumentacja

## Przegląd projektu

Meeting Planner to lekka aplikacja webowa napisana w Django, ułatwiająca organizację spotkań: tworzenie propozycji terminów, głosowanie uczestników oraz wybór ostatecznego terminu i miejsca. Projekt skupia się na prostocie użytkowania i szybkim wdrożeniu lokalnym.

## Główne funkcje

- Tworzenie spotkań — definiowanie tytułu, opisu i propozycji terminów.
- Głosowanie uczestników — użytkownicy mogą wskazać preferowane terminy.
- Agregacja wyników — podsumowanie głosów i wybór najlepszego terminu.
- Proste UI i szablony — gotowe strony: dashboard, tworzenie, wyniki, głosowanie.
- Integracje dla produkcji — konfiguracje dla WhiteNoise i Gunicorn (serwer WSGI).

## Stos technologiczny

- Python >= 3.11
- Django >= 5.2
- django-htmx (interaktywność bez pełnego SPA)
- Gunicorn (serwer produkcyjny)
- psycopg2-binary (obsługa PostgreSQL dla produkcji)
- WhiteNoise (serwowanie statycznych plików w produkcji)

Pliki konfiguracyjne warte uwagi: `pyproject.toml`, `kiedy_gdzie/settings.py`, oraz (dla prostego uruchomienia) `db.sqlite3` w katalogu projektu.

## Cel dokumentacji

Ta dokumentacja ma pomóc deweloperom i współpracownikom szybko zrozumieć cele projektu, przegląd funkcji i podstawowy sposób uruchomienia aplikacji lokalnie. Zawiera także wskazówki, gdzie szukać konfiguracji produkcyjnych.

## Krótki start (lokalnie)

Najprostszy sposób uruchomienia deweloperskiego (zakładając SQLite):

```zsh
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt  # lub `pip install -e .` gdy używasz pyproject.toml
python manage.py migrate
python manage.py runserver
```

Dla środowiska produkcyjnego: skonfiguruj zmienne środowiskowe, użyj PostgreSQL (psycopg2-binary) oraz uruchom aplikację pod Gunicorn, ewentualnie za reverse-proxy.
